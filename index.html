Update Github 'repo structure' so we can demonstrate the use case:

  • developer makes a change to their feature branch
    1. developer pulls master branch from Github using Git in a terminal: 
      git pull origin master or git clone * repo url *
    2. developer switches to the desired branch 
      git checkout -b feature1
    3. changes are made to the new feature1 branch and committed 
      git commit -m "enter what was changed here"
    4. changes are pushed from the local repo to the remote repo feature1 branch 
      git push origin -feature1
  • developer submits a pull request from Github page
  • lead developer reviews/approves/denys pull request from Github page
    1. Once a pull request has been made, the maintainer will see a message on the screen like this.
  • code merged into master when pull request is approved
    1. feature1 branch is deleted in remote repo 
      this is done after the pull request approval
